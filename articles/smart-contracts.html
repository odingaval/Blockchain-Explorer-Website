Smart Contracts: Code as Law

By Maya Patel Posted on June 2, 2023 12 min read
smart contracts solidity ethereum

What Are Smart Contracts?

Smart contracts are self-executing contracts with the terms of the agreement between buyer and seller being directly written into lines of code. They were first proposed by Nick Szabo in 1994 and became practical with the advent of blockchain technology.

Key Characteristics:

Self-executing

Automatically execute when conditions are met

Tamper-proof

Cannot be changed once deployed

Decentralized

Run across blockchain nodes

Transparent

Code and transactions are visible

How Smart Contracts Work

Smart contracts follow a simple "if/when...then..." logic that is deployed to a blockchain. When predetermined conditions are met, the contract executes the corresponding actions.

Smart Contract Lifecycle

Phase 1
Contract Creation

Developer writes and deploys contract code to blockchain

Phase 2
Contract Address

Contract gets a unique blockchain address

Phase 3
Interaction

Users and other contracts can interact with it

Phase 4
Execution

Conditions trigger automatic execution

Writing Smart Contracts in Solidity

Solidity is the most popular language for writing Ethereum smart contracts. It's a statically-typed language with syntax similar to JavaScript.

Basic Smart Contract Example

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    // State variable stored on blockchain
    uint storedData;
    
    // Function to set the value
    function set(uint x) public {
        storedData = x;
    }
    
    // Function to get the value
    function get() public view returns (uint) {
        return storedData;
    }
}

Key Solidity Concepts

State Variables

Permanently stored in contract storage

uint public count = 0;

Functions

Executable units of code

function increment() public {
    count += 1;
}

Modifiers

Change function behavior

modifier onlyOwner() {
    require(msg.sender == owner);
    _;
}

Events

Logs for external consumers

event ValueChanged(uint newValue);

function set(uint x) public {
    storedData = x;
    emit ValueChanged(x);
}

Gas and Optimization

Every operation in Ethereum costs gas, which is paid in ETH. Optimizing gas usage is crucial.

Gas Costs for Common Operations

Operation Gas Cost Description
SSTORE (new value) 20,000 Writing to storage
SSTORE (existing value) 2,900 Updating storage
CALL 700 Basic call operation
CREATE 32,000 Creating a new contract

Smart Contract Security

Security is critical as smart contracts handle real value. Common vulnerabilities include:

Reentrancy Attacks

When a contract makes an external call before resolving its state, allowing recursive calls.

// Vulnerable code
function withdraw() public {
    require(balances[msg.sender] > 0);
    (bool success, ) = msg.sender.call{
        value: balances[msg.sender]
    }("");
    require(success);
    balances[msg.sender] = 0;
}

Integer Overflow/Underflow

When arithmetic operations exceed maximum or minimum values.

// Before Solidity 0.8.0
uint8 x = 255;
x++; // x is now 0 (overflow)

Security Best Practices

  • Use checked math (Solidity 0.8.0+) or SafeMath
  • Follow the checks-effects-interactions pattern
  • Use established libraries like OpenZeppelin
  • Get professional audits for production contracts
